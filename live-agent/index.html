LiveAgent is an MCP server that exposes a Phoenix LiveView app's socket assigns, component tree, telemetry, and browser DOM to AI coding tools — in real time, from the running BEAM. Think live_debugger, but as an instrument panel Claude can read and drive.

DEV-ONLY

act_as · impersonation

One call lands the session authenticated as any user — built for testing multi-tenant org isolation. Four independent locks; none compile in prod.

  • build gate (only: :dev)
  • env gate (:dev/:test)
  • app-supplied closure
  • Drive toggle
VERIFY

Assertions & diffs

Turn "read the state and judge it" into an explicit verdict — and a real anchor for verify loops.

  • expect_assign
  • expect_no_errors
  • screenshot_diff
  • scratchpad_compare
⚠ SECURITY POSTURE — LiveAgent is dev-only and localhost-guarded. It returns 403 to any non-127.0.0.1 request and must never be mounted on a production endpoint.
[ DEPLOY ]

Three steps to link

// STEP 01

Add the dep

# mix.exs {:live_agent, "~> 0.2", only: :dev}
// STEP 02

Plug the endpoint

# endpoint.ex if Code.ensure_loaded?( LiveAgent) do plug LiveAgent end
// STEP 03

Point Claude at it

// .mcp.json "live-agent": { "type": "http", "url": "…/live_agent/mcp" }

RUN mix phx.server — CLAUDE CODE CONNECTS OVER HTTP WHILE YOUR APP IS RUNNING. NO SEPARATE PROCESS.
