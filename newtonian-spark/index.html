The idea

A prompt is a graph, not a string.

Each node is a discrete piece of logic — a persona, a constraint, a piece of context, a branch. Edges define how they compose. Because the design is structured, the platform can validate it, version it, and compile it to model-ready instructions on every change.

Prompt as a text blob

  • One giant string, edited by hand
  • Guardrails copy-pasted across agents
  • A renamed variable silently breaks production
  • "Is it good?" is a vibe check

Prompt as architecture

  • Typed nodes with explicit dependencies
  • Reusable skills — change once, propagate
  • An enforced input contract per version
  • Diagnostics catch errors before deploy
What you get

A studio for agent logic.

Visual graph studio
Design on a canvas
Persona, constraint, context, conditional, skill, memory, tool, evaluation and output nodes — composed visually instead of buried in prose.
Variable contract
Inputs that can't drift
Write {variable} anywhere. The studio discovers every variable, maps producers and consumers, and flags what's undefined or unused.
Diagnostics
A linter for prompts
Cycle detection, floating nodes, missing persona/output, conflicting instructions — surfaced live, before anything ships.
Skill registry
Reusable, not copy-pasted
Skills, policies, schemas and memory templates are first-class assets. Link them into many graphs; update the source once.
Compiler
Model-ready output
Assembles nodes in dependency order and compiles to provider-specific formats, with token and cost estimates.
Deployment
Pinned & versioned
Publish a graph version to dev, staging or production. Applications fetch a pinned version and inject runtime variables.
The canvas

Read an agent at a glance.

Typed blueprint nodes, conditional diamonds with yes/no routing, and inline variable highlighting — review an agent you didn't write the way you'd review code.
